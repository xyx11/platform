## 测试步骤

### 测试 1：创建并保存流程
1. 访问 http://localhost:3001/#/system/workflow-designer
2. 点击"新建"按钮
3. 从左侧工具栏拖拽元素：
   - 开始事件
   - 用户任务（命名为"经理审批"）
   - 结束事件
4. 连接元素
5. 按 Ctrl+S 保存
6. 输入流程名称：请假流程
7. 验证保存成功

### 测试 2：关联表单
1. 保存流程后，点击"关联表单"按钮
2. 选择启动表单（需先创建表单）
3. 为"经理审批"任务选择表单
4. 设置处理人类型和值
5. 保存配置

### 测试 3：部署流程
1. 点击"部署流程"按钮
2. 确认部署
3. 部署成功后点击"关联表单"
4. 配置表单

### 测试 4：发起流程
1. 点击"发起流程"按钮
2. 选择已部署的流程
3. 输入业务 Key：请假 -20240101-001
4. 填写启动表单
5. 点击"发起流程"
6. 验证流程实例创建成功

### 测试 5：自动保存
1. 修改流程
2. 观察右上角保存状态
3. 等待 3 秒，验证自动保存
4. 看到"自动保存成功"提示

### 测试 6：版本历史
1. 保存流程后，点击"历史"按钮
2. 查看版本列表
3. 点击"加载"恢复到旧版本
4. 点击"删除"删除旧版本

### 测试 7：快捷键
1. 按 Ctrl+? 或点击"帮助"按钮
2. 查看快捷键列表
3. 测试各快捷键功能

## API 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| /system/workflow/save | POST | 保存流程 |
| /system/workflow/deploy | POST | 部署流程 |
| /system/workflow/definition/list | GET | 获取流程定义列表 |
| /system/workflow/form/config | POST | 保存表单配置 |
| /system/workflow/form/start/{key} | GET | 获取启动表单 |
| /system/workflow/instance/start | POST | 启动流程实例 |
| /system/workflow/form-definition/list | GET | 获取表单定义列表 |

## 数据库表

- `workflow_form_binding` - 表单绑定关系
- `form_definition` - 表单定义
- `act_re_*` - Flowable 流程定义表
- `act_ru_*` - Flowable 运行时表

## 注意事项

1. 确保后端服务运行在 http://localhost:8082
2. 确保前端服务运行在 http://localhost:3001
3. 确保已登录并有相应权限
4. 确保 `workflow_form_binding` 表已创建
5. 确保 Flowable 引擎已集成
