## 1. SAP NetWeaver BPM 概述

### 1.1 什么是 SAP NetWeaver BPM

SAP NetWeaver Business Process Management (BPM) 是 SAP 公司提供的一套完整的业务流程管理解决方案，它允许企业：

- 建模和自动化端到端业务流程
- 集成异构系统
- 监控和优化流程性能
- 支持人工审批和自动服务调用

### 1.2 核心特性

| 特性 | 描述 |
|------|------|
| **流程建模** | 使用 BPMN 2.0 标准进行可视化流程设计 |
| **流程编排** | 协调人工任务和自动服务 |
| **业务规则** | 集成业务规则引擎进行决策 |
| **表单管理** | 动态生成用户任务界面 |
| **流程监控** | 实时跟踪流程实例状态 |
| **集成能力** | 与 SAP 和非 SAP 系统无缝集成 |

### 1.3 与本系统的对应关系

| SAP NetWeaver BPM | 本系统模块 |
|-------------------|-----------|
| Process Designer | 流程设计器 (`/system/workflow-designer`) |
| Process Repository | 流程定义 (`/system/workflow-definition`) |
| Form Builder | 流程表单 (`/system/workflow-form`) |
| Process Viewer | 流程实例监控 (`/system/process-instance`) |
| Task Center | 任务管理 (`/system/task`) |
| Process Statistics | 工作流统计 (`/system/workflow/stats`) |

## 2. BPMN 2.0 基础

### 2.1 核心元素

#### 2.1.1 事件 (Events)

事件表示流程中发生的事情，用圆形表示。

| 图标 | 名称 | 描述 | 使用场景 |
|------|------|------|----------|
| ○ (绿色) | **开始事件** | 触发流程启动 | 每个流程必须有至少一个 |
| ○ (红色) | **结束事件** | 标记流程或路径结束 | 每个流程应该有至少一个 |
| ⏰ | **定时事件** | 基于时间触发 | 定时启动、超时处理 |
| ✉️ | **消息事件** | 接收/发送消息 | 与外部系统通信 |
| ⚠️ | **错误事件** | 处理异常情况 | 错误捕获和异常处理 |

#### 2.1.2 活动 (Activities)

活动表示需要执行的工作，用圆角矩形表示。

| 图标 | 名称 | 描述 | 使用场景 |
|------|------|------|----------|
| 👤 | **用户任务** | 需要人工处理的任务 | 审批、填写信息 |
| ⚙️ | **服务任务** | 自动调用的服务 | API 调用、数据处理 |
| 📜 | **脚本任务** | 执行脚本代码 | 数据转换、逻辑计算 |
| 📨 | **发送任务** | 发送消息 | 发送邮件、消息通知 |
| 📥 | **接收任务** | 接收消息 | 等待外部响应 |
| ✋ | **手动任务** | 手动执行的操作 | 线下操作、电话确认 |

#### 2.1.3 网关 (Gateways)

网关控制流程的分支和合并，用菱形表示。

| 图标 | 名称 | 描述 | 使用场景 |
|------|------|------|----------|
| ✖️ (X) | **排他网关 (XOR)** | 只能选择一个路径 | 条件判断、单选 |
| ➕ (+) | **并行网关 (AND)** | 同时执行所有路径 | 并行处理、会签 |
| ⭕ (O) | **包容网关 (OR)** | 可以选择一个或多个路径 | 多选条件 |
| 📅 | **事件网关** | 基于事件触发选择 | 竞争事件处理 |

#### 2.1.4 数据对象

| 图标 | 名称 | 描述 |
|------|------|------|
| 📄 | **数据对象** | 流程中使用的数据 |
| 🗄️ | **数据存储** | 持久化存储的数据源 |

#### 2.1.5 子流程

| 类型 | 描述 | 使用场景 |
|------|------|----------|
| **嵌入式子流程** | 包含在主流程内的子流程 | 流程模块化、分组 |
| **事务子流程** | 支持事务回滚的子流程 | 需要原子性的操作 |
| **事件子流程** | 由事件触发的子流程 | 异常处理、中断处理 |

### 2.2 连接类型

| 类型 | 线型 | 描述 |
|------|------|------|
| **顺序流** | 实线箭头 | 定义流程执行顺序 |
| **消息流** | 虚线圆圈 | 跨池/泳道的消息传递 |
| **关联** | 点线 | 连接数据和活动 |

```

│          │                                │                 │
│  工具栏   │           画布区域             │   属性面板       │

│          │                                │                 │
│ - 事件    │     [开始]→[任务]→[结束]        │ - 元素 ID       │
│ - 任务    │                                │ - 名称          │
│ - 网关    │                                │ - 监听器        │
│ - 数据    │                                │ - 表单          │

```

### 4.2 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl + N` | 新建流程 |
| `Ctrl + O` | 打开文件 |
| `Ctrl + S` | 保存流程 |
| `Ctrl + Z` | 撤销 |
| `Ctrl + Y` | 重做 |
| `Delete` | 删除选中元素 |
| `+` | 放大 |
| `-` | 缩小 |
| `0` | 适应屏幕 |

### 4.3 操作流程

1. **新建流程**
   - 点击「新建」按钮或按 `Ctrl + N`
   - 在弹出的对话框中填写流程基本信息

2. **添加元素**
   - 从左侧工具栏拖拽元素到画布
   - 或点击元素后在画布中心创建

3. **连接元素**
   - 选中起始元素
   - 按住 `Ctrl` 拖拽到目标元素

4. **配置属性**
   - 选中元素
   - 在右侧属性面板填写配置

5. **验证流程**
   - 点击「验证」按钮检查流程完整性
   - 确保有开始事件和结束事件

6. **部署流程**
   - 点击「部署」按钮
   - 填写部署信息
   - 提交到流程引擎

## 5. 流程变量管理

### 5.1 变量类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **String** | 字符串 | "张三", "财务部" |
| **Number** | 数字 | 100, 3.14 |
| **Boolean** | 布尔值 | true, false |
| **Date** | 日期时间 | 2024-01-15 10:30:00 |
| **Object** | JSON 对象 | {"name": "张三", "age": 30} |
| **List** | 列表 | ["北京", "上海", "广州"] |

### 5.2 变量作用域

| 作用域 | 描述 | 生命周期 |
|--------|------|----------|
| **流程变量** | 整个流程实例共享 | 流程实例开始到结束 |
| **任务变量** | 仅在当前任务可用 | 任务创建到完成 |
| **子流程变量** | 在子流程内共享 | 子流程实例周期 |
| **执行变量** | 在执行路径上共享 | 执行路径周期 |

### 5.3 变量配置

在流程设计器中配置变量：

```json
{
  "variables": [
    {
      "name": "applicantName",
      "label": "申请人姓名",
      "type": "String",
      "required": true,
      "defaultValue": ""
    },
    {
      "name": "amount",
      "label": "报销金额",
      "type": "Number",
      "required": true,
      "defaultValue": 0
    },
    {
      "name": "approvalRequired",
      "label": "是否需要审批",
      "type": "Boolean",
      "required": false,
      "defaultValue": true
    }
  ]
}
```

### 5.4 变量表达式

使用表达式引用变量：

| 场景 | 表达式 | 示例 |
|------|--------|------|
| **条件判断** | `${variable}` | `${amount > 1000}` |
| **赋值** | `${variable = value}` | `${result = approved}` |
| **方法调用** | `${object.method()}` | `${stringUtils.toUpperCase(name)}` |
| **数学运算** | `${a + b}` | `${total = price * quantity}` |

## 6. 监听器配置

### 6.1 执行监听器 (Execution Listener)

监听流程执行过程中的事件。

| 事件 | 触发时机 | 使用场景 |
|------|----------|----------|
| **start** | 活动/流程开始时 | 初始化变量、记录日志 |
| **end** | 活动/流程结束时 | 清理数据、发送通知 |
| **take** | 顺序流被选取时 | 记录路径选择 |

#### 配置示例

```xml
<bpmn:sequenceFlow id="flow1" sourceRef="start" targetRef="task1">
  <bpmn:extensionElements>
    <camunda:executionListener event="take">
      <camunda:script><![CDATA[
        execution.setVariable("pathSelected", "approved");
      ]]></camunda:script>
    </camunda:executionListener>
  </bpmn:extensionElements>
</bpmn:sequenceFlow>
```

### 6.2 任务监听器 (Task Listener)

监听用户任务的生命周期事件。

| 事件 | 触发时机 | 使用场景 |
|------|----------|----------|
| **create** | 任务创建时 | 设置处理人、初始化数据 |
| **assignment** | 任务分配时 | 动态分配处理人 |
| **complete** | 任务完成时 | 数据验证、结果处理 |
| **delete** | 任务删除时 | 清理关联数据 |

#### 配置示例

```xml
<bpmn:userTask id="approvalTask" name="经理审批">
  <bpmn:extensionElements>
    <camunda:taskListener event="create">
      <camunda:script><![CDATA[
        task.setAssignee("manager_" + execution.getVariable("deptId"));
      ]]></camunda:script>
    </camunda:taskListener>
    <camunda:taskListener event="complete">
      <camunda:script><![CDATA[
        execution.setVariable("approvalTime", new java.util.Date());
      ]]></camunda:script>
    </camunda:taskListener>
  </bpmn:extensionElements>
</bpmn:userTask>
```

### 6.3 监听器实现方式

| 方式 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| **脚本** | 直接写脚本代码 | 简单快捷 | 难以复用和维护 |
| **表达式** | 使用 EL 表达式 | 简洁易读 | 功能有限 |
| **Java 类** | 实现监听器接口 | 功能强大、可复用 | 需要编译部署 |
| **Delegate 类** | 实现 JavaDelegate | 适合服务任务 | 需要 Java 开发 |

## 8. 最佳实践

### 8.1 流程建模规范

1. **命名规范**
   - 流程 ID：使用小写字母和连字符，如 `leave-approval-process`
   - 任务名称：使用「动词 + 名词」格式，如「提交申请」、「经理审批」
   - 变量名：使用驼峰命名，如 `applicantName`、`approvalAmount`

2. **流程结构**
   - 每个流程只允许有一个开始事件
   - 必须有至少一个结束事件
   - 避免循环依赖
   - 使用子流程进行模块化

3. **网关使用**
   - XOR 网关用于互斥条件
   - AND 网关用于并行处理
   - 分支条件要明确且不重叠
   - 汇聚网关要与分支网关配对

### 8.2 性能优化

1. **流程设计**
   - 避免过长的流程链
   - 减少并行分支数量
   - 使用异步服务任务

2. **变量管理**
   - 只存储必要的变量
   - 大对象使用外部存储
   - 及时清理不用的变量

3. **历史数据**
   - 配置历史数据归档策略
   - 定期清理已完成的历史
   - 分离实时数据和历史数据

### 8.3 错误处理

1. **异常捕获**
   ```
   [任务] → [错误事件] → [异常处理子流程] → [结束事件]
   ```

2. **超时处理**
   ```
   [任务] + [定时边界事件] → [超时处理路径]
   ```

3. **重试机制**
   - 服务任务配置重试次数
   - 失败后进入异常处理流程
   - 记录失败原因和上下文

### 8.4 安全建议

1. **权限控制**
   - 任务处理人必须验证
   - 敏感数据加密存储
   - 操作日志完整记录

2. **数据验证**
   - 所有输入数据进行验证
   - 防止 SQL 注入
   - 防止 XSS 攻击

3. **流程安全**
   - 限制流程发起权限
   - 敏感流程多级审批
   - 定期审计流程执行

## 附录

### A. 常见问题解答

**Q: 流程部署后如何修改？**
A: BPMN 文件是只读的，需要修改时重新部署新版本。流程引擎会自动使用最新版本处理新发起的流程实例。

**Q: 如何查看流程执行历史？**
A: 在「流程实例监控」页面可以查看流程执行路径、任务完成情况和变量变更记录。

**Q: 任务处理人如何动态设置？**
A: 使用任务监听器在 `create` 事件中动态设置处理人，或者在流程变量中预定义处理人表达式。

**Q: 如何实现会签功能？**
A: 使用并行网关创建多个并行分支，每个分支对应一个审批人，最后使用并行网关汇聚。
